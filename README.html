1. Project Overview

Wavely is a modern news platform developed with Flask as part of the Coding Dojo Python Bootcamp final project (January 2025). The application provides a complete ecosystem for news articles management, user interactions, and content moderation with AI-powered analytics.

Key Purpose: Wavely enables users to create, share, and interact with news articles while providing rich features like article favoriting, commenting, and AI-powered analysis of content bias and objectivity.

3.3 Blueprint Architecture

Wavely implements a modular architecture using Flask Blueprints, each responsible for a specific functional area of the application. This approach promotes clean separation of concerns, maintainability, and code organization.

Blueprint Responsibility
autentication User registration, login, session management
noticias News article creation, editing, viewing, deletion
comentarios Comment system on news articles
favoritos Article favoriting functionality
likes_noticias Article liking system
likes_comentarios Comment liking system
report_noticias Article reporting for moderation
report_comentarios Comment reporting for moderation
administrador Admin dashboard and moderation functions
usuarios User profile management

4. Database Models

Wavely uses SQLAlchemy ORM with a relational database structure. The core models and their relationships are:

4.1 Core Models

  • Usuario: User accounts and authentication information
  • Noticia: News articles with content and metadata
  • Comentario: User comments on articles
  • Favorito: Tracks user favorite articles
  • Like_Noticia: Records user likes on articles
  • Like_Comentario: Records user likes on comments
  • Report_Noticia: Tracks reported articles for moderation
  • Report_Comentario: Tracks reported comments for moderation

4.2 Key Relationships

Usuario (1) --< Noticia (many)          # A user can create many articles
Noticia (1) --< Comentario (many)       # An article can have many comments
Usuario (1) --< Comentario (many)       # A user can create many comments
Usuario (1) --< Favorito (many)         # A user can favorite many articles
Noticia (1) --< Favorito (many)         # An article can be favorited by many users
Usuario (1) --< Like_Noticia (many)     # A user can like many articles
Noticia (1) --< Like_Noticia (many)     # An article can be liked by many users
Usuario (1) --< Like_Comentario (many)  # A user can like many comments
Comentario (1) --< Like_Comentario (many) # A comment can be liked by many users

5. AI Integration

One of Wavely's standout features is its integration with OpenAI's API to provide automated analysis of news content. The system implements:

5.1 Content Analysis

News articles are analyzed asynchronously using Celery tasks to provide:

  • Content categorization: Determines the article's topic and type
  • Sentiment analysis: Evaluates the emotional tone of the article
  • Summary generation: Creates concise summaries of longer articles

5.2 Bias Detection

The platform includes a sophisticated bias detection system that:

  • Analyzes articles for political or ideological bias
  • Scores content on an objectivity scale
  • Identifies potentially misleading content patterns
  • Provides transparency to users about potential biases
Implementation Note: These AI analyses are performed asynchronously using Celery tasks, ensuring the main application remains responsive while complex analysis happens in the background.

6. Installation and Setup

6.1 Prerequisites

  • Python 3.8+ installed
  • MySQL Server
  • Redis Server (for Celery task queue)
  • OpenAI API key (for AI features)

6.2 Environment Setup

  1. Clone the repository: 
    git clone [repository-url]
cd 00_codigo
  2. Create and activate a virtual environment: 
    python -m venv venv
source venv/bin/activate  # On macOS/Linux
venv\Scripts\activate     # On Windows
  3. Install dependencies: 
    pip install -r requirements.txt
  4. Create a .env file based on the example.env template: 
    USER=your_username
PASSWORD=your_password
OPENAI=your_openai_api_key
DATABASE='db_noticias'
HOST_DATABASE=localhost
USER_DATABASE=root
PORT=5001
SECRET_KEY="your_secret_key"
ADMINS=['admin@wavely.com']
UPLOAD_FOLDER=static/uploads
ALLOWED_EXTENSIONS=png,jpg,jpeg,gif,mp4,mov,avi,webm
DEBUG=True

6.3 Database Setup

  1. Create a MySQL database named 'db_noticias'
  2. Update the database connection details in your .env file
  3. The application will automatically create tables when first run

6.4 Running the Application

Main Application

python server.py

Celery Worker (in a separate terminal)

celery -A flask_app.tasks.celery_app worker --loglevel=info
Important: Make sure Redis server is running before starting the Celery worker.

7. Key Workflows

7.1 Article Creation and Analysis

  1. User authenticates into the system
  2. User navigates to "Add Article" and submits content
  3. System saves the article to the database
  4. Two Celery tasks are triggered asynchronously:
    • analisis_noticia_task: Categories and summarizes the article
    • analisis_sesgo_task: Analyzes the article for potential bias
  5. AI analysis results are updated in the database
  6. Users can view analysis results on the article page

7.2 User Interaction Flow

  1. Users can browse the news feed
  2. Interact with articles through:
    • Commenting
    • Liking
    • Favoriting for later reading
    • Reporting inappropriate content
  3. View personalized feeds based on interests and favorites

7.3 Moderation Workflow

  1. Users report inappropriate content
  2. Reports are logged in the database
  3. Administrators can review reported items in the admin dashboard
  4. Administrators can take actions:
    • Dismiss reports
    • Edit content
    • Remove content
    • Issue warnings to users

8. File Upload System

Wavely includes a secure system for handling multimedia uploads for news articles:

  • Supported formats: PNG, JPG, JPEG, GIF, MP4, MOV, AVI, WEBM
  • Security features:
    • Filename sanitization
    • UUID-based unique filenames to prevent overwrites
    • Extension validation
  • Storage: Files are saved to the configured UPLOAD_FOLDER (static/uploads by default)
# Example of upload handling from noticias_controller.py
if 'archivo_multimedia' in request.files:
    file = request.files['archivo_multimedia']
    if file and file.filename != '' and allowed_file(file.filename):
        filename = secure_filename(file.filename)
        unique_filename = f"{uuid.uuid4()}_{filename}"
        file_save_path = os.path.join(UPLOAD_FOLDER, unique_filename)
        file.save(file_save_path)
        file_path = f"/static/uploads/{unique_filename}"

9. Security Features

  • Password Security: Bcrypt hashing with appropriate work factors
  • Session Management: Secure session cookies and server-side session data
  • Route Protection: Login_required decorator prevents unauthorized access
  • Input Sanitization: Form validation and secure handling of user input
  • File Upload Validation: Extension checking and secure file naming
  • Environment Variables: Sensitive configuration stored in .env file

10. Development and Extension

10.1 Adding New Features

The modular Blueprint architecture makes it easy to extend functionality:

  1. Create a new model in flask_app/models/
  2. Create a new controller in flask_app/controllers/
  3. Register the new Blueprint in flask_app/__init__.py
  4. Create templates for the new feature

10.2 Development Best Practices

  • Code Style: Follow PEP 8 conventions for Python code
  • Template Structure: Use template inheritance from base.html
  • Database Changes: Update models and ensure relationships are properly defined
  • Error Handling: Use Flash messages for user feedback
  • Security: Validate all inputs and use login_required decorator on protected routes
